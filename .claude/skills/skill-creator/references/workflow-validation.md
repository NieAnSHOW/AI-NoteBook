# 验证模式

在工作流中嵌入验证步骤，确保执行准确性。

## 验证时机

| 时机 | 验证内容 | 标记 |
|------|----------|------|
| 前置验证 | 检查必需条件 | **MANDATORY** |
| 过程验证 | 验证中间结果 | **CRITICAL** |
| 结果验证 | 确认最终输出 | Checklist |

## 验证模板

```markdown
#### 验证清单

**前置验证** (开始前):
- **MANDATORY**: 检查必需条件
- 确认输入数据完整

**过程验证** (关键步骤后):
- **CRITICAL**: 立即验证输出格式
- 记录中间结果

**结果验证** (任务结束后):
- 运行验证脚本
- 确认最终输出符合预期
```

## 示例

### 数据处理验证

```markdown
#### 验证清单

**前置验证**:
- **MANDATORY**: 检查输入文件存在
- 确认文件格式正确

**过程验证**:
- **CRITICAL**: 数据清洗后检查记录数
- 验证转换结果无空值

**结果验证**:
- 运行 `python scripts/validate_output.py`
- 确认输出文件大小合理
```

## 适用场景

- 数据质量保证
- 关键业务流程
- 多步骤处理
- 外部系统集成
