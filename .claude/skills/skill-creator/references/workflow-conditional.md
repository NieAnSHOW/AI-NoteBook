# 条件工作流模式

适用于需要根据条件选择不同路径的任务。

## 模式说明

条件工作流根据输入或中间结果选择不同的执行路径。

## 模板

```markdown
1. 确定[判断维度]：
   **[条件A]?** → 执行"[工作流A]"
   **[条件B]?** → 执行"[工作流B]"

2. [工作流A]: [步骤列表]
3. [工作流B]: [步骤列表]
```

## 示例

### 内容修改流程

```markdown
1. 确定修改类型：
   **创建新内容?** → 执行"创建工作流"
   **编辑现有内容?** → 执行"编辑工作流"

2. 创建工作流:
   - 收集需求
   - 生成模板
   - 填充内容
   - 验证输出

3. 编辑工作流:
   - 加载现有内容
   - 识别修改点
   - 应用修改
   - 验证变更
```

### 文件处理流程

```markdown
1. 确定文件类型：
   **PDF 文件?** → 执行"PDF 处理流程"
   **图片文件?** → 执行"图片处理流程"
   **文本文件?** → 执行"文本处理流程"

2. PDF 处理流程: [步骤]
3. 图片处理流程: [步骤]
4. 文本处理流程: [步骤]
```

## 适用场景

- 多类型输入处理
- 用户选择分支
- 错误恢复路径
- 功能模式切换
